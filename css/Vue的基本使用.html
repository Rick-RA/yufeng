# computed和watch

computed有缓存，data不变则不会重新计算。

watch如何深度监听？

watch在监听引用类型时：

  1. 如果是给引用类型重新赋值，那么可以监听到引用类型的改变 。重新赋值： obj = {name: 'rick'}
  2. 如果是给引用类型中的key赋值，那么监听不到，需要改用深度监听。 key赋值：obj.name = 'rick'
watch: {
  obj: {
    handler(oldVal, val) {
      console.log(oldVal, val)
    },
    deep: true   // 深度监听  
  }
}

watch使用深度监听时，拿不到oldVal，拿到的oldVal和val是同一个值，并且是最新的值。

# v-show 和 v-if

v-show是通过控制display: none属性来控制元素

v-if是操作DOM，对DOM进行创建和删除操作。

如果是频繁的操作，建议使用v-show，能够提高性能。如果是控制某个组件，v-show有的生命周期函数不会调用。v-if会调用，例如created

# eventBus

一般多用于兄弟组件之间进行通信

  1. 可以先定义一个全局event

    export default const event = new Vue()

  2. 在一个组件中监听事件

    import event from './event.js'
event.$on('customeEvent', this.handleEvent)

  3. 另外一个组件中触发事件

    import event from './event.js'
event.$emit('customeEvent', val)

  4. 解绑事件

    beforeDestroy() {
  event.$off('customeEvent', this.handleEvent)
},

eventbus的缺点：

有时候业务复杂时，多次触发同一事件。或者有些业务导致监听组件已经释放了，触发时无法生效。

比如： 我们想通过A组件进入B组件时在B中触发某个事件E。但是当我们从C组件打开B时，也会触发这个事件。

另外一个就是有时候A组件已经被销毁了，但是我们从其他组件进入B的时候，也会触发事件E，但是可能我们自己并不知道A组件已经被销毁了。

# 生命周期

以上是vue官网提供的生命周期的流程图。一共有8个钩子函数。分别对应：

  • beforeCreate
  • created
  • beforeMount
  • mounted
  • beforeUpdate
  • updated
  • beforeDestroy
  • destroyed

# 父子组件的生命周期：

加载：

  1. 父 -> beforeCreate
  2. 父 -> created
  3. 父 -> beforeMount
  4. 子 -> beforeCreate
  5. 子 -> created
  6. 子 -> beforeMount
  7. 子 -> mounted
  8. 父 -> mounted

更新：

  1. 父 -> beforeUpdate
  2. 子 -> beforeUpdate
  3. 子 -> updated
  4. 父-> updated

销毁：

  1. 父 -> beforeDestroy
  2. 子 -> beforeDestroy
  3. 子 -> destroyed
  4. 父 -> destroyed

# $nextTick

  1. Vue是异步渲染的
  2. data改变之后，DOM不会立即渲染，页面渲染时会将data的修改做整合，多次data修改只会渲染一次.
  3. $nextTick会在DOM渲染之后被触发，以获取最新DOM节点

# mixin

多个组件有相同的逻辑，抽离出来。

mixin并不是完美的解决方案，会有一些问题。

Vue3提出的Composition API旨在解决这些问题。

// 定义一个混入对象
var myMixin = {
  created: function () {
    this.hello()
  },
  methods: {
    hello: function () {
      console.log('hello from mixin!')
    }
  }
}

// 定义一个使用混入对象的组件
var Component = Vue.extend({
  mixins: [myMixin]
})

var component = new Component() // => "hello from mixin!"

当组件和混入对象含有同名选项时，这些选项将以恰当的方式进行“合并”。

  • 数据（data函数）中的对象在内部会进行递归合并，在冲突时以组件数据优先。
  • 同名钩子函数合并成一个数组，都将被调用。先调用混入的钩子，再调用组件的钩子
  • 值为对象的选项，例如methods, components和directives，将被合并为同一个对象。两个对象键名冲突时，取组件对象的键值对。

也可以自定义选项合并策略，通过Vue.config.optionMergeStrategies;

mixin的问题：

  • 变量来源不明，不利于阅读
  • 多mixin可能会造成命名冲突
  • mixin和组件之间可能出现多对多的关系，复杂度较高

# 几个常见问题的整理

# v-for为何要使用key

必须是key，且不能是index和random

diff算法中tag和key来判断是否是同一个节点，如果是则认为改节点没有改变，不再进行深层次的比较。

通过设置key可以减少渲染次数，提升渲染性能

# 双向数据绑定v-model的实现原理

input元素的value = this.name

绑定input事件，this.name = $event.target.value

data更新触发re-render

使用函数，在不同地方使用的时候实际上是执行了函数，不会造成影响。
