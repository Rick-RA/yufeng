# Vue的响应式原理

Vue是通过Object.defineProperty来重新定义data中声明的属性，并实现他们的get和set放来来进行监听。

Object.definePropety(target, 'name', {
  get: function() {
    return value
  },
  set: function(newValue) {
    if(newValue !== value) {
      value = newValue
      updateView(value)
    }
  }
})

在set方法中，如果监听到数据变化，则调用更新视图的操作。

如果需要监听的data不是简单的值类型，而是引用类型，则底层实现的时候需要遍历递归遍历引用类型，然后对其值类型的key进行Object.definePropety监听操作。

# Object.definePropety的缺点

  • 深度监听时，需要递归到底，一次性计算量大
  • 无法监听新增/删除属性,需要使用Vue.set和Vue.delete来实现，否则无法更新视图
  • 除了数组赋值外，无法原生监听数组的各种操作，需要其他方法单独处理

# 数组的监听

const oldArrayprototype = Array.prototype
const arrayProto = Object.create(oldArrayprototype)
['push', 'pop', 'shift', 'unshift', 'splice'].forEach(methodName => {
  arrayProto[methodName] = function() {
    updateView();
    oldArrayprototype[methodName].call(this, ...arguments)
  }
})

# 虚拟DOM

使用虚拟DOM的原因和前提：

  • 手动操作DOM非常耗费性能
  • JS计算非常高效

定义：虚拟DOM是一个有JS模拟的树状结构的JS对象，它上的节点和真是的DOM树中的节点一一对应。

<div id="div1" class="container">
  <p>
    vdom
  </p>
  <ul style="font-size: 20px">
    <li>a</li>
  </ul>
</div>

上述代码转化成虚拟DOM后结构如下：

{
  	tag: 'div',
  	props: {
    	className: 'container',
    	id: 'div1'  
  	},
    children: [
      {
        tag: 'p',
        children: 'vdom'
      },
      {
        tag: 'ul',
        props: {
          style: 'font-size: 20px'
        },
        children: [
          {
            tag: 'li',
            children: 'a'
          }
        ]
      }
    ]  
    
    
    
}

# vdom diff算法

默认DOM树进行比较，时间复杂度是O(n^3)

diff算法优化时间复杂度到O(n)

比较逻辑：

  • 只比较同一层级，不做跨层比较
  • tag不相同，则直接删掉重建，不再深度比较
  • tag和key，两者都相同，则认为是相同节点，不再深度比较。

# with语法

  • 改变{}内自由变量的查找规则，当做obj属性来查找
  • 如果找不到匹配的obj属性，就会报错
  • with要慎用，它打破了作用域规则，易读性变差

html是标签语言，不能执行一些逻辑操作等，但是可以直接被浏览器渲染。vue的模板不能直接作为html模板被浏览器渲染。所以本质上vue的模板是被编译成Javascript代码。

模板编译为render函数，执行render函数，并返回vnode

基于vnode再执行patch和diff

使用webpack vue-loader，会在开发环境下编译模板。

vue组件可以用render代替template

# Vue组件是如何渲染和更新的

  1. 解析模板为render函数（或开发环境已完成， vue-loader）
  2. 触发响应式，监听data属性getter setter
  3. 执行render函数，生成vnode，patch(elem, vnode)
  4. 如果修改data，触发setter（此前在getter中已被监听）
  5. 重新执行render函数，生成vnode
  6. patch(elem, vnode)
