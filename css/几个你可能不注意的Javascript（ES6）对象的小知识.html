# 几个你可能不注意的ES6新增的小知识

# 自有属性的枚举顺序

ES5中没有定义对象属性的顺序，由各Js引擎厂商自行决定。ES6中严格规定了对象的自有属性被枚举是返回的顺序。这会影响到Object.getOwnPropertyNames()方法以及Reflect.ownKeys()返回属性的方式，Object.assign()方法处理属性的顺序也将随之改变。

基本规则是：

  1. 所以数字按升序排序
  2. 所有字符串按它们被加入对象的顺序排序
  3. 所有symbol键按照它们被加入对象的顺序排序。
var obj = {
  a: 1,
  0: 1,
  c: 1,
  2: 1,
  b: 1,
  1: 1
}
obj.d = 1

console.log(Object.getOwnPropertyNames())    // "012acbd"

对于for-in循环，由于并非所有厂商都遵循相同的实现方式，因此仍未指定一个明确的枚举顺序；

而Object.keys()和JSON.stringify()都知名与for-in使用相同的枚举顺序才能正确运行。

# ES6新增了setPrototypeOf()来改变对象的原型

接收两个参数：

参数一： 被改变原型对象

参数二：替代第一个参数原型的对象

# ES6在对象中引入了super关键字

在对象中，super的作用等价于Object.getPrototypeOf(this).例如：

let friend = {
  getGreeting(){
    return super.getGreeting() + 'hi'
  }
}

// 但是非简写形式下不能使用super
let friend = {
  getGreeting: funtion(){   
    return super.getGreeting() + 'hi' // 此种写法会报错
  }
}
