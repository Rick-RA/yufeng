# BFC

# Formatting context

Formatting context是W3C CSS2.1规范中的一个概念。它定义了页面中的一块渲染区域的渲染规则。它决定了元素如何定位，以及和其他元素的关系和相互作用。

  • BFC - Block formatting context （块级格式化上下文）
  • IFC - Inline formatting context （内联格式化上下文）

BFC决定了元素如何对其内容进行定位，以及与其他元素的关系和相互作用。BFC提供了一个环境，HTML按照一定规则在这个元素中进行布局。一个环境中的元素不会影响其他环境中的布局。

比如：浮动会形成BFC，浮动元素内部的子元素主要受到该元素的影响。两个浮动元素之间是互不影响的。

# 如何触发BFC

  • float的值不为none
  • overflow的值不为visible
  • display的值为table-cell, table-caption, inline-block中的任一种
  • position的值不为relative和static

# 规则

  • 盒子从顶部开始垂直排列
  • 两个相邻盒子的距离由外边距（margin）决定
  • BFC相邻的盒子之间的垂直外边距重叠
  • 每个盒子的左外边与容器的左边接触，即使存在浮动也是如此。除非盒子建立了新的格式化上下文
  • 形成BFC的区域不会与float box重叠
  • 计算BFC高度时，浮动子元素也参与计算

# 应用

  1. 同一个BFC下外边距会发生折叠
  2. BFC可以包含浮动的元素（清除浮动）
  3. BFC可以阻止浮动元素覆盖
