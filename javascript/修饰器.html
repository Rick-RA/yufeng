# 修饰器

# 类的修饰

# 是一个函数，用来修饰类的行为

# 修饰器的行为的改变时在代码编译时发生的，而不是运行是。

# 第一个参数就是要修饰的类 function testable(target){}, target就是被修饰的类。

# 如果一个参数不够，可以在修饰器外再封装一层函数

# 子主题 5

# 方法的修饰

# 修饰器不仅可以修饰类，还可以修饰类的属性。

# 此时接收三个参数（1、所要修饰的目标对象 2、所要修饰的属性名 3、该属性的描述对象）

# 为什么修饰器不能用于函数

# 修饰器只能用于类和方法，不能用于函数，因为存在函数提升

# core-decorators.js

# 是一个第三方模块，提供了几个常见的修饰器，可以更好地理解修饰器

# @autobind

  • 使得方法中的this对象绑定原始对象

# @readonly

  • 属性或方法不可写

# @override

  • 检查子类方法是否覆盖了父类的同名方法，如果不正确报错

# @deprecate(@deprecated)

  • 在控制台显示一条警告，表示该方法将废除

# @suppressWarnings

  • 修饰器抑制decorated修饰器导致的console.warn()调用，但异步代码发出的调用除外

# 在一个对象中混入另外一个对象的方法

# Trait

# 效果与mixin类似，但是提供了更多功能，比如防止同名方法的冲突，排除混入某些方法、为混入的方法起别名等。
