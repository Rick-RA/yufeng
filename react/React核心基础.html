# React核心基础

# 元素渲染

React元素是一个不可变对象，更改元素的唯一方式是创建一个全新的元素。

function tick() {
  const element = (
    <div>
      <h1>Hello, world!</h1>
      <h2>It is {new Date().toLocaleTimeString()}.</h2>
    </div>
  );
  ReactDOM.render(element, document.getElementById('root'));}

setInterval(tick, 1000);

React只更新它需要更新的部分。

# 组件

组件类似于Javascript函数，接受任意入参(props)，并返回一个用于描述页面真是内容的React元素。

  • 函数组件

    function Welcome(props) {
  return (
  	<h1>
    	hello, {props.name}
    </h1>
  )
}

  • class组件

    class Welcome extends React.Component {
  render () {
    return (
      	<h1>
    			hello, {this.props.name}
    		</h1>
    )
  }
}

    上述两种写法是等效的。

# Props

组件无论是使用函数声明还是class声明，都不能修改自身的props。

纯函数： 多次调用下，相同的入参始终返回相同的结果。

所有的React组件都必须像纯函数一样保护它们的props不被更改。

# State

将函数组件转换为class组件:

  1. 创建一个同名的ES6 class，并且继承React.Component
  2. 添加一个空的render()方法
  3. 将函数体移动到render方法中
  4. 在render()方法中使用this.props替换props
  5. 删除剩余的空函数声明

state的使用：

  • 不要直接修改state，使用setState()；构造函数中是唯一可以使用this.state赋值的地方。直接修改state，代码不会重新渲染组件。

  • State的更新可能是异步的 出于性能考虑，React可能会把多个setState() 调用合并成一个调用 例如，如下代码可能导致无法更新计数器

    this.setState({
  counter: this.state.counter + this.props.increment
})

    为解决上述问题，setState()接收的参数是一个函数，而不是一个对象。例如：

    // Correct
this.setState(function(state, props) {
  return {
    counter: state.counter + props.increment
  };
});

  • State的更新会被合并

  • 数据是向下流动的

    组件可以选择把他的state作为props向下传递到他的子组件中。

# 事件处理

  • React事件的命名采用小驼峰

  • 使用JSX语法时，你需要传入一个函数作为事件处理函数，而不是一个字符串。例如：

    <button onClick={activateLasers}>
  Activate Lasers
</button>

    在 React 中另一个不同点是你不能通过返回 false 的方式阻止默认行为。你必须显式的使用 preventDefault

    在Javascript中，class的方法默认不会绑定this。所以你需要手动绑定this, 或者使用箭头函数

    class Toggle extends React.Component {
  constructor(props) {
    super(props);
    this.state = {isToggleOn: true};

    // 为了在回调中使用 `this`，这个绑定是必不可少的
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    this.setState(state => ({
      isToggleOn: !state.isToggleOn
    }));
  }

  render() {
    return (
      <button onClick={this.handleClick}>
        {this.state.isToggleOn ? 'ON' : 'OFF'}
      </button>
    );
  }
}

ReactDOM.render(
  <Toggle />,
  document.getElementById('root')
);

    使用箭头函数：

    class LoggingButton extends React.Component {
  // 此语法确保 `handleClick` 内的 `this` 已被绑定。
  handleClick = () => {
    console.log('this is:', this);
  }

  render() {
    return (
      <button onClick={this.handleClick}>
        Click me
      </button>
    );
  }
}

  • 向事件处理程序传递参数

    <button onClick={(e) => this.deleteRow(id, e)}>Delete Row</button>
<button onClick={this.deleteRow.bind(this, id)}>Delete Row</button>

# 列表和key

列表中的每个元素应该有一个唯一的key

万不得已可以使用元素的索引index作为key

如果列表项目的顺序可能会发生变化，我们不建议使用索引index来作为key值。因为这样可能会导致性能问题和引起一起组件状态的问题。

元素的key应该就近的数组上下文中才有意义。

key只是在兄弟节点之间必须唯一。但不需要是全局唯一的。

JSX允许在大括号中嵌入任何表达式
